```typescript
/**
 * Phase 2: Scalar Field Type Matrix
 *
 * Parameterised undo/redo round-trip test for every scalar field type.
 * Each case: change → undo (assert original) → redo (assert changed).
 *
 * Requirements: MATRIX-02 (text), MATRIX-03 (number), MATRIX-04 (boolean),
 * MATRIX-05 (radio), MATRIX-06 (select), MATRIX-07 (date).
 *
 * Failing tests are NOT skipped — they become Phase 8 input.
 */

import { mountAutosaveHarness, scalarSchema } from "@test-helpers/autosaveHarness";
```

Define a `SCALAR_CASES` table with these exact entries:

| field    | initial                                    | changed                                    |
|----------|--------------------------------------------|--------------------------------------------|
| "text"   | "initial text"                             | "updated text"                             |
| "num"    | 42                                         | 99                                         |
| "bool"   | false                                      | true                                       |
| "radio"  | "option-a"                                 | "option-b"                                 |
| "select" | "value-1"                                  | "value-2"                                  |
| "date"   | new Date("2026-01-01T00:00:00.000Z")       | new Date("2026-06-15T00:00:00.000Z")       |

Use `describe.each(SCALAR_CASES)("$field field", ({ field, initial, changed }) => { ... })` to parameterise. Inside the describe.each block, write a single `it` that:

1. Mounts the harness: `mountAutosaveHarness({ defaultValues: scalarSchema(), config: { debounceMs: 50, debug: true }, undo: { enabled: true, hotkeys: false } })`
2. Wraps the body in `try { ... } finally { harness.unmount(); }`
3. Asserts the starting value matches `initial`: for Date fields, compare `.getTime()` timestamps; for all others, use `toBe(initial)`.
4. Calls `harness.change(field, changed)` and asserts the value is now `changed`.
5. Calls `harness.undo()` and asserts the value reverted to `initial`.
6. Calls `harness.redo()` and asserts the value restored to `changed`.

For Date-specific assertions, create a helper function `assertValue` at the top of the file:

```typescript
function assertValue(actual: unknown, expected: unknown, label: string): void {
  if (expected instanceof Date) {
    // Value equality via timestamp
    expect(actual).toEqual(expected);
    // Type preservation — if this fails, Date degraded to string (Phase 8 bug)
    expect(actual).toBeInstanceOf(Date);
  } else {
    expect(actual).toBe(expected);
  }
}
```

Use `assertValue(harness.form.getValues(field), initial, "after undo")` and `assertValue(harness.form.getValues(field), changed, "after redo")` for all assertions.

The `beforeEach`/`afterEach` block follows the canonical pattern:
```typescript
beforeEach(() => { jest.useFakeTimers(); });
afterEach(() => { jest.runOnlyPendingTimers(); jest.useRealTimers(); });
```

**What NOT to do:**
- Do NOT render DOM elements (<input>, <select>, etc.). The harness works at the form-values level.
- Do NOT import `act`, `useForm`, `useRhfAutosave`, `waitFor`, or `waitForCondition`.
- Do NOT use `it.skip`, `xit`, or `xdescribe`. Failing tests stay red for Phase 8.
- Do NOT call `jest.advanceTimersByTime` directly. The harness's `waitForIdle` is the timing API (but Phase 2 tests are synchronous — no timing needed).
- Do NOT import from `src/testing/`.
- Do NOT use `toBe` for Date comparison — use `toEqual` (Jest handles Date objects correctly with toEqual) plus `toBeInstanceOf(Date)` for type preservation.
- Do NOT use `toBeFalsy()` for booleans — use `toBe(false)` explicitly.
